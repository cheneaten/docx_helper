# DOCX 结构化规则文件

## 你的任务（只做这一件事）

分析下方文档的内容结构，输出一个 **纯 JSON 数组**。除此之外不要输出任何其他内容。

## 输出格式（严格遵循）

一个合法的 JSON 数组，每个元素是一个标题插入指令。**不要用 ``` 包裹，不要有任何说明文字**，**不要换行缩进**：

```json
[{"level":1,"text":"一、引言","anchor":"本文旨在探讨"}]
```

## 字段说明

| 字段       | 类型     | 说明                                      |
| -------- | ------ | --------------------------------------- |
| `level`  | 整数 1-6 | 标题级别。一级=1，二级=2，依此类推                     |
| `text`   | 字符串    | 完整的标题文本（含编号前缀）                          |
| `anchor` | 字符串    | **锚点**：原文中该标题所辖内容的**第一段段首的前 4~6 个连续汉字** |

## 标题命名规范

| 级别  | 格式        | 示例                     |
| --- | --------- | ---------------------- |
| H1  | 中文数字 + 顿号 | `一、引言` `二、相关工作` `三、方法` |
| H2  | 括号中文数字    | `（一）研究背景` `（二）实验设计`    |
| H3+ | 阿拉伯数字 + 点 | `1. 实验设置` `2. 数据分析`    |

## 锚点选取规则（最重要）

**锚点是整个方案正确执行的关键。** 请按以下流程操作：

1. 找到该标题所辖内容的第一句话
2. **只取前 4~6 个连续汉字**，不含标点符号
   - ✅ 正确：`"由于系统底层限制一般"`（去掉标点，取 6 字）
   - ❌ 错误：`"由于系统底层限制，一般是无法进行"`（太长，含标点）
3. 在文档中搜索这个锚点，如果能**唯一匹配**则使用
4. 如果匹配到多处，加长到 8~10 字直到唯一

## 必须遵守的规则

- ✅ 输出**一行**合法 JSON，不加任何换行和空格
- ✅ 第一个标题必须是 H1
- ✅ 必须逐级递进，不能跳级（H1→H2→H3，不能 H1→H3）
- ✅ 按文档顺序排列
- ❌ 不要在 JSON 前后加任何文字、代码块、解释
- ❌ 不要在末尾总结或复述标题列表
- ❌ 不要编造锚点

## 示例

文档原文：

```
本文旨在探讨人工智能在医疗领域的最新进展。
近年来随着深度学习技术的突破，医学影像分析取得显著成果。
另一方面自然语言处理技术也被广泛应用于电子病历分析。
```
